# План робіт: палітра, карусель, хедер і бренд

Документ для послідовної реалізації (без прізвища в назві бренду, з логотипом, сильнішою палітрою та акуратним хедером).

---

## Фаза A — Бренд і неймінг (пріоритет: високий)

**Проблема:** назва містить прізвище; немає логотипу; тексти дублюються в metadata, футері, інтро.

**Завдання**

1. **Узгодити з PO нейтральну назву магазину** (приклади напрямку, не фінал): *Resonance*, *Stage & Tone*, *Octave*, *Phantom Audio* — без прізвищ і без «Sound» як обов’язкового суфікса, якщо хочеться відходу від поточного шаблону.
2. **Логотип**
   - Варіант A: SVG-марк у `components/brand/Logo.tsx` (геометричний знак + wordmark), один колір + варіант для темного фону.
   - Варіант B: wordmark лише типографікою (кастомний lockup), якщо не робимо іконку зараз.
3. **Оновити всі входження старої назви**
   - `app/layout.tsx` — `metadata.title`, `metadata.openGraph`, `metadata.description`.
   - `components/layout/Header.tsx` — лінк і видимий текст.
   - `components/layout/Footer.tsx` — copyright і бренд без прізвища (наприклад «Октава»).
   - `components/home/HomeIntro.tsx` — рядок з брендом у шапці секції.
   - `app/admin/*`, `robots.ts`, `sitemap` — лише якщо є згадки.
   - `.env.example` / коментарі в коді — за потреби.

**Критерій готовності:** одна узгоджена назва скрізь; у UI є хоча б SVG-логотип або чіткий типографічний lockup; прізвище розробника ніде не фігурує як назва магазину.

## Фаза B — Кольорова система і «динаміка» (пріоритет: високий)

**Проблема:** майже монохром (black + zinc + рідкі accent); готичний темний тон ок, але візуально плоско.

**Завдання**

1. **Токени в `app/globals.css` (`:root`)**
   - Фон: не суцільний `#000`, а глибина — наприклад `bg-zinc-950` + ледь видимий градієнт або noise (CSS `background-image` з прозорим шумом / radial mesh).
   - **Primary accent** (кліки, focus, активні стани): один насичений колір (наприклад бурштин/мідь `#d4a574` або холодний лайм `#b4e068` — обрати одну лінію, не змішувати).
   - **Secondary accent** (промо, hover карток): другий відтінок у тій же сім’ї або комплементарний з обмеженим використанням.
   - **Border / glass:** замінити всюди `border-white/10` на семантичні `border-border` з токена (наприклад `hsl(..., 0.12 opacity)`).
2. **Розширити `tailwind.config.ts`**
   - `theme.extend.colors`: `brand`, `surface`, `muted`, `accent`, `accent-secondary`.
   - За потреби `backgroundImage` для `hero-mesh`, `card-glow`.
3. **Точково оживити сторінки**
   - Головна: `HomeIntro`, зона каруселі — легкий mesh або radial spotlight за токенами.
   - Кнопки primary/secondary у хедері та CTA — з нового accent.
   - Картки каталогу / адмін таблиці — hover з кольоровим border або тінню accent (не переборщити).

**Критерій готовності:** мінімум 4 семантичні кольори в токенах; Tailwind використовує `brand/*`; головна й каталог візуально «не прісні» без відходу від темної теми.

## Фаза C — Хедер: вирівнювання, структура, мобільна навігація (пріоритет: високий)

**Проблема:** `justify-between` з трьома блоками (лого | nav | кошик) — на десктопі навігація **не по центру** візуально; немає логотипу; немає мобільного меню.

**Завдання**

1. **Сітка хедера (рекомендований патерн)**
   - Триколоночна сітка `grid-cols-3` однакової ширини колонок: `[лого + мобільна кнопка] [центрована nav] [кошик + utility]`.
   - Або: лого зліва, **абсолютно центрована** nav (`absolute left-1/2 -translate-x-1/2`), справа кошик — з `pointer-events` і `z-index`, щоб не ламати кліки.
2. **Компоненти**
   - `Header.tsx` — рефактор під обрану сітку; додати `Logo` + назву або лише lockup.
   - `MobileNav.tsx` (client) — sheet або full-screen overlay для `md:hidden`, ті самі лінки + кошик не дублювати двічі.
3. **Посилання `/about`**
   - Якщо сторінки немає — прибрати з nav або створити мінімальну `app/about/page.tsx` (щоб не було 404).

**Критерій готовності:** на `md+` навігація візуально по центру відносно в’юпорту; логотип зліва; на мобільному — згорнуте меню; кошик завжди доступний.

## Фаза D — Карусель 3D: поліпшення (пріоритет: середній)

**Файл:** `components/home/ProductCarousel3D.tsx`

**Ідеї (обрати 2–4)**

1. **Візуал:** легкий `box-shadow` / glow у кольорі accent на «передній» картці (обчислити найближчу до 0° панель або використати фіксований highlight на активному індексі).
2. **Інерція:** після відпускання drag додати сповільнення (velocity → damping), замість миттєвої зупинки.
3. **Керування:** кнопки «Назад / Вперед» + опційно клавіатура (`ArrowLeft` / `ArrowRight`).
4. **Доступність:** `prefers-reduced-motion` вже частково є — розширити: статична сітка 2×2 замість 3D.
5. **Глибина:** легкий `scale` для дальніх карток (за кутом від камери) — обережно з продуктивністю.

**Критерій готовності:** зміни помітні в UX; без регресій на мобільному; Lighthouse не падає критично.
